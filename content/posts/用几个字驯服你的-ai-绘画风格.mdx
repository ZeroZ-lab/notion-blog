在玩 AI 画图一段时间之后，你会发现一个很有趣的现象：

- 明明只是随手加了几句描述，例如「透明探索版」「儿童科普风」「赛博蓝图感」，
- 画面立刻从“随机好看”变成了“有固定味道”。

这些词表面上只是几个字，实质上却是一个**高维风格向量的压缩表示**——你用极低的字数，唤起了模型内部一整个「视觉空间」的偏好：材质、光影、构图、情绪、用途，都一起被推向了某个方向。


我们可以把这个过程，抽象成一个非常程序员友好的式子：


```plain text
Style = f(材质 M, 光影 L, 构图 C, 情绪 E, 功能 U)
```


也就是说：

- 「风格」不是一个标签，而是一个**函数的返回值**；
- 每一次你说「XXX 版」，其实都是在给 (M, L, C, E, U) 这五个参数赋值；
- 这些参数的不同组合，构成了你在用的所有视觉语言。

接下来，我们就按这五个维度，把这个抽象真正“拆开、枚举、可用”。


---


## 1. 风格不是标签，而是五个维度的坐标


很多人理解风格的方式是：

- 「这是赛博朋克风」
- 「这是儿童插画风」
- 「这是技术蓝图风」

这种说法没错，但太模糊了。对人脑来说勉强够用，对写 prompt 来说就有点粗糙。


如果你把视角稍微抽象一格，会看到一个更清晰的结构：

> 一张图的“风味”，可以拆成五个独立的决策：
> 1. **材质（Material）**：东西“像是用什么做的”？玻璃、金属、木头、纸、软糖、能量体？
> 2. **光影（Lighting）**：光从哪来？冷暖？对比？自然还是人工？
> 3. **构图（Composition）**：你站在哪个视角看？俯视、剖面、爆炸、时间轴？
> 4. **情绪（Emotion）**：它想让人感觉什么？探索、紧张、怀旧、治愈、史诗？
> 5. **功能（Use）**：它是用来干什么的？教学、海报、结构说明、系统架构图？
>

你之前起的那些名字：

- 透明探索版
- 霓虹夜行版
- 剖面楼层版
- 儿童科普版
- 未来蓝图版

其实都可以看作是：

> 在五个维度上，各选了 1–2 个值，然后打包成一个好记的名字。

这意味着：

- 风格是「可枚举、可组合、可编程」的；
- 你完全可以给自己设计一门“风格 DSL（小语法）”；
- Prompt 不再是堆砌词，而是给函数参数赋值。

下面我们就开始“枚举值域”。


---


## 2. 材质（M）：东西是“什么做的”


材质决定了：

- 细节长什么样（螺丝？木纹？网格？折痕？）
- 高光 / 阴影怎么表现（硬反光 vs 柔漫反射）
- 观众的“触感想象”（冰冷 / 温暖 / 柔软 / 粘稠）

常用的材质值，可以这样列一个表：

- **透明 / 玻璃感**：玻璃、水晶、折射、透视
- **金属 / 机甲**：金属壳、铆钉、机械结构、硬边
- **塑料 / 软胶 / 软糖**：玩具感、Q 版、高光、圆润
- **木质 / 原木**：木头积木、桌游棋子、温暖纹理
- **纸张 / 折纸**：纸模型、折纸、纸板、褶皱
- **布料 / 织物**：布纹、刺绣、挂毯、褶皱阴影
- **石头 / 岩石 / 混凝土**：纪念碑、遗迹、粗糙表面
- **霓虹 / 光线当材质**：线条本身发光、字体就是光
- **信息 / 界面 / HUD**：UI 面板悬浮在物体/空间上
- **生物组织 / 解剖感**：像器官、神经、血管网络
- **颗粒 / 胶片 / 噪点**：随处可见的颗粒、粗糙 film 感
- **液体 / 流体**：水、墨水、液态金属、流淌的形状
- **烟雾 / 云雾**：雾化、不完全实体、模糊边界
- **数字网格 / 线框**：wireframe、网格面、低多边形
- **发光体 / 能量体**：能量球、光束、粒子云

每选一个材质值，其实就是在告诉模型：

> “请把世界的所有表面都往这个方向偏移。”

例如：

- 「水晶解构版」≈ 材质锁在“玻璃 / 水晶 + 爆炸视图”的组合上。

---


## 3. 光影（L）：光从哪来，气氛就往哪走


光影比人们想象的要重要得多：

- 内容一样，光一改，故事就换了。
- 同一座城，日出是希望感；霓虹是躁动；大雾是迷失。

可以把光影抽象成这些常用枚举值：

- **日光 / 自然光**：柔和、真实、适合“默认世界观”
- **日出 / 黄昏光**：金色、长阴影、情绪偏「纪实 + 诗意」
- **夜景 / 霓虹**：城市灯牌、反射水面、强对比、偏潮流
- **极光 / 梦幻光**：绿色/紫色渐变、天空发光、非日常
- **冷光 / 科技蓝**：偏蓝青色、冷静、实验室 / 科技感
- **暖光 / 治疗系**：暖黄、柔软、室内、咖啡馆、温馨
- **背光 / 剪影**：主体变成剪影、强调轮廓与张力
- **点光源 / 舞台光**：聚光灯、焦点极强、戏剧性
- **高对比 / 硬光**：阴影边界清晰，张力大
- **低对比 / 雾化**：画面整体柔，信息藏在氛围里
- **单色 / 黑白**：抽掉颜色，强调结构与明暗
- **电影感 / 胶片光**：暗角、漏光、小范围过曝
- **超现实光源**：地面发光、悬浮光柱、UI 当光源

选定光影之后，相当于给这张图选了一个「世界观的气候」。


例如：

- 「霓虹夜行版」≈ 夜景 + 霓虹 + 潮流情绪
- 「极光梦境版」≈ 极光天空 + 柔和渐变 + 梦境情绪

---


## 4. 构图（C）：你从哪里看这个世界？


构图决定的是：

- 观众站在哪里；
- 信息怎么排布；
- 哪个部分是主角，哪个是背景。

我们可以把构图的常见模式也当作枚举值：

- **正视 / 平视**：像人站在眼前看，适合肖像、单体主体
- **俯视 / 上帝视角**：鸟瞰、地图感、全局信息
- **仰视 / 崇高感**：夸张建筑、史诗、庄严、压迫
- **斜 45° / 等距视角**：游戏感、城市模拟器风格
- **剖面视图**：物体 / 建筑被“切开”，内部一览
- **爆炸视图**：各个部件分离、悬浮，用于结构说明
- **流程图 / 节点-连线**：逻辑关系、系统流动、箭头
- **时间轴构图**：从左到右或从下到上是时间
- **对称构图**：中央轴线、仪式感
- **局部特写**：超近距离只看一个部位
- **全景 / 宽画幅**：横向信息很多、世界观展开
- **分屏 / 多格漫画**：画面被切成多格，每格一个瞬间
- **楼层剖面 / 娃娃屋**：像把建筑一切开，看见每层房间
- **中景 + 小人物**：环境巨大、角色渺小，适合“城市孤独”
- **中心辐射 / 放射型**：从中心往外扩散的能量或结构
- **焦点引导线**：所有线条/元素指向一个关键点

想想「剖面楼层版」：

- 构图维度被锁死在「楼被剖开 + 多层 + 娃娃屋视角」上，
- 其他材质/光影/情绪可以变，但“结构阅读方式”已经定了。

---


## 5. 情绪（E）：图要让人“感觉成什么”


内容一样，情绪可以完全不同：

- 同一个系统结构图，
    - 做成「儿童科普版」：圆角、温和、无压力；
    - 做成「灾难预警版」：红色、告警图标、压迫感。

情绪这维可以这样枚举：

- **探索 / 好奇**：邀请你进去看看，偏“科研 + 冒险”
- **科普 / 安心**：清晰、规范、友好、可靠
- **紧张 / 危机**：高对比、红色、告警、时间不多的感觉
- **孤独 / 冷静**：冷色调、小人物、大环境
- **梦境 / 超现实**：不合理组合、漂浮、魔幻
- **怀旧 / 记忆**：褪色、胶片感、轻微模糊
- **温暖 / 治愈**：暖色、柔边、安全空间
- **史诗 / 崇高**：巨大尺度、仰视、光柱、阶梯
- **混乱 / 过载**：信息过多、重叠、故障感
- **冷静 / 专业**：克制、规整、蓝灰色调
- **游戏化 / 轻松**：卡通、表情夸张、奖励/金币元素
- **神圣 / 仪式感**：神殿、祭台、光束
- **未来感 / 预见**：时间感 + 科幻感 + 不确定性
- **灾难后 / 末世**：破碎、灰尘、残骸、空城

这维度和光影、构图互动非常强：

- 「城市孤独版」 = 小人物 + 大环境 + 冷色调 + 安静构图
- 「记忆闪回版」 = 胶片颗粒 + 模糊 + 暖偏色 + 时间感

---


## 6. 功能（U）：这张图“拿来干嘛的”


功能维度常常被忽略，但对做教学 / 产品文档的人来说，它是核心之一：

- 决定信息密度
- 决定标注多少
- 决定抽象 vs 具象

可以这样枚举：

- **概念海报 / Hero 图**：主角强、信息少、情绪重
- **结构说明 / 原理讲解**：剖面 / 爆炸 + 标签 + 注释
- **流程教学 / 步骤讲解**：Step1/2/3，适合教程
- **系统架构图**：模块 + 箭头 + 边界（很适合 AI / 后端）
- **儿童科普图**：少字、多图、易懂、大图标
- **对比 / 前后变化**：before/after、优劣对照
- **仪表盘 / 数据可视化**：图表 + 卡片 + 组件
- **产品界面 Mockup**：像 App / Web 截图
- **Logo / 图标级简化**：极简、几何、单一元素
- **叙事插画 / 场景故事**：一张图讲一个情节
- **教科书插图**：规范、结构正确、标注精细
- **演示文稿配图**：简洁、一眼看懂、适合投影
- **概念设计草图**：多个小视角、小注释、探索性强

当你说「未来蓝图版 · 系统架构图」时，其实已经非常具体了：

- 功能：用于展示系统结构；
- 构图：偏蓝图 / 结构框图；
- 材质：像在蓝图纸上的线条；
- 情绪：偏专业、冷静、未来感。

---


## 7. 把这一切收束回那个函数


现在再回看最开始那句：


```plain text
Style = f(材质 M, 光影 L, 构图 C, 情绪 E, 功能 U)
```


你可以让它变成**一个可操作的“风格签名”**，例如：


```plain text
Style = (
  材质: 透明 + 信息 HUD,
  光影: 冷光 + 高对比,
  构图: 剖面楼层,
  情绪: 探索 + 科普,
  功能: 系统架构图
)
```


然后，把它压缩成一句人类可读的风格名：

> 「透明探索版 · 剖面楼层架构图」

在 Prompt 里，则可以展开为一长串英文描述：


```plain text
a clean isometric cutaway diagram,
transparent layers showing internal modules,
holographic UI overlays on each floor,
cool blue sci-fi lighting, high contrast,
clear labels for each component,
professional system architecture infographic
```


从这一步开始，你做的事情就不再是“凭感觉堆词”，而是：

- 先在脑中给 M、L、C、E、U 五个参数赋值；
- 然后用一个自己习惯的命名方式压缩；
- 再在 Prompt 中按模型的语言展开。

你得到的是一整套**参数化的风格系统**，而不是一堆零散的 prompt 灵感。
